3）项目亮点介绍
Agent-Hub 解决的是企业协作里“AI 能生成内容，但过程不可控、状态不可见、结果难落地”的问题。传统流程里，用户需要在 IM、文档、PPT、云盘之间反复切换：先描述需求，再人工整理 Brief，再生成演示稿，再上传分享，还要持续询问进度。Agent-Hub 把这条链路收敛成一句飞书消息触发的闭环流程。
项目的核心亮点是“多端可控的 Agent 协作中台”：用户在飞书提出任务，Agent 自动规划步骤，关键写入和分享动作进入审批，执行进度在飞书、电脑 Dashboard、手机 Dashboard 三端同步，最终交付可打开的 PPTX 和飞书云文档链接。它不是只做一次模型调用，而是把任务拆解、审批、执行、产物、事件、恢复都纳入同一套工程化流程。
落地价值主要体现在三点：第一，把文档/PPT 生产从人工多工具切换变成分钟级自动流水线；第二，人从“重复执行者”变成“目标提出者和关键风险审批者”；第三，所有状态变化、审批决议和产物都可追踪，便于演示、复盘和后续扩展到更多企业工作流。
4）AI 亮点介绍
项目使用了多种高阶 AI 工程技巧。首先是结构化输出规划：OpenAIModelGateway 使用 JSON mode 生成 PlanBlueprint，并用 Pydantic 校验、Skill 白名单和模板 fallback 保证稳定性，避免模型自由输出导致流程不可执行，见 model_gateway.py:449、model_gateway.py:525。
其次是“AI 规划 + 人类审批 + Skill 执行”的分工方式。AI 负责理解自然语言、选择任务 profile、生成步骤计划、产出 Brief/SlideSpec/PPT；人负责在计划、写入、分享等关键风险点审批；系统负责幂等、状态机、事件存储、失败恢复和多端同步。这种模式比完全自动化更适合企业场景，因为它兼顾效率和控制权。
模型选型上，项目采用 OpenAI 兼容接口作为统一 LLM Gateway，可接入智谱 GLM 等模型；规划失败时降级到确定性模板，保证 Demo 稳定。知识检索部分采用向量检索 + BM25 + RRF 融合，兼顾语义相关性和关键词召回，见 pipeline.py:70、pipeline.py:126、hybrid_ranker.py:16。
AI Agent 能力也不止生成文本。LLMAgent 支持 ReAct 推理循环，通过 Thought、Action、Observation 调用工具，并记录 ReActTrace；ReflectionAgent 使用 LangGraph 实现检索、评估、重写、生成的容错链路；PromptGuard 使用规则 + LLM 语义检测做注入防御。这些设计体现了项目不是“套壳聊天机器人”，而是具备路由、工具、检索、安全和可观测性的 Agent 工程系统。
